## 챕터 1~3 정리

### 입사 첫날 느끼기

- AI를 활용해서 회사 프로젝트에 대한 파악을 해보자
  - 프로젝트 분석
  - 컨벤션 확인
  - 구조 파악
  - 일관적이지 못한, 정책이 제대로 정리되지 못한 부분들을 파악해서 리팩토링 방안도 뽑아보자.
    - 이부분을 첫 회의, 스크럼 때 언급해보고 얼라인을 맞춰보자.

#### 정리

실제로 저도 최근에 이직을 하고 회사에 입사해서 프로젝트를 파악하는데 AI의 도움을 매우 많이 받으면서 진행했습니다.
클로드 코드 CLI를 주로 사용했고 소속 파트의 도메인에 대해 API 별로 정리하고 각 기능을 설명해달라고 하고, 어떤 동작을 하는것인지 알려달라고 했었습니다.

- AI 요청 시 주의점
  - 가이드라인을 만들자
  - 광역적인 요청을 했더니 매우 무의미한, 생산적이지 못한 개선을 하더라.
    1. 일을 시키는것도 잘해야한다.
    2. 두루뭉술한 가이드라인대로 리팩토링 해주세요. 한들 우리라고 잘할까?
    3. 지침을 구체적으로 만들자.
  - 단계별 지침으로 리팩토링 전략을 수정하자.
    1. Assembler 서비스로 서비스 묶기.
    2. 레거시를 느끼면서 개선해보기

#### 정리

확실히 AI 한테는 요구할 때 명확한 range와 요구사항을 정확히 전달하는게 유의미한 결과 차이를 낳는게 느껴진다.
리팩토링을 요청할 때도

```
객체지향 기반의 코드로 리팩토링 해줘.
```

보다는

```
객체지향 기반의 코드로 리팩토링할 꺼야.
객체지향은 객체가 행동을 기반으로 역할을 맡게 되고 객체 간 협력과 소통(메서드)를 통해 동작하는 방향이야.

이때 유지보수를 위해 1. 영향범위, 2. 확장성, 3. 의존성 이렇게 3가지 기준을 지키면서 개발해야해.

이를 위해 5가지 원칙이 있어.
1. SRP: 설명~
2. OCP: 설명~
...

이렇게 5가지 원칙을 기반으로 리팩토링 할거야.
1번 리팩토링 단계로 SRP 원칙에 위배되는 클래스를 확인하고 개선을 진행하자.
```

이렇게 주저리 써줘야 더 맥락과 의도를 이해하고 해준다고 느꼈다.

### 상품 목록 느끼기

- 상품 목록 페이지에 대해서
  - 최근 찜 수
  - 최근 주문 수
- 두 기능 추가 요청이 왔다.

이런 요청이 왔을 때 어떻게 해야하지?를 생각하자.

- 필요한 기능이 뭐지?
- 필요한 행동이 뭐지?

AI한테 이렇게 요청했더니 이런 좋은점/문제점/개선점이 있네를 인지하자.

#### 정리

확실히 이제 회사에서 업무를 하게되면 지라티켓에서 요구사항부터 클로드에게 묻고 시작하게 된다.
이런 수동적인 작업조차 번거럽다고 느껴지니 지라티켓이 등록되면 자동으로 AI에게 요청이 가도록 MCP 서버를 사람들이 구성하려고 하는거같다.
지라티켓이 내앞으로 따졌을 때 자동으로 요구사항 정리, 구현 계획만 우선적으로 만들도록 해놔도 훨씬 편할거같다. 특히 이런 프롬프트는 피쳐/버그 등에 따라 거의 유사할것이므로 스킬 등록을 통해 쉽게 처리할 수 있을거같다.

## 스터디 블로깅 주제 정리

### 주제

- AI 페어 프로그래밍으로 백엔드 과제 재도전

### 개요

백엔드 과제 2~3개를 강의에서 제공하는 내용을 바탕으로 레거시 AI 협업 전략으로 풀어보고, 결과를 비교하고자 합니다.

스터디 강의를 기반으로 과제를 AI와 협력하여 구현하고 개선하는것이며 AI를 어떻게 써야 하는지 검증하는것을 목표로 합니다.

각 과제마다 AI가 잘한것 / 못한 것 / 내가 개입한 것을 기록해서 정량적으로 분석하고 기록하고자 합니다.

강의에서 말하는 요구사항 및 코드 느끼기 기법을 따라하며 AI를 사용하고자 합니다.

1. 요구사항 느끼기
2. 아키텍처 설계 + 스켈레톤 구조 잡기 + 핵심기술 결정
3. AI와 함께 구현
   - 이때, 아래 2가지 구현 완성도에 대해 비교
   - 단순히 필요한 기능 구현 나열 후 요청하는것
   - 구체적인 설계와 구조(틀)를 잡아놓고 기능 구현 요청하는것
   - 리뷰 -> 수정 -> 리뷰 ... 를 반복
   - 강의 내용 최대한 녹이기
4. 코드 느끼기
   - AI가 놓치는 한계점 인식하기
   - 내가 설정한 설계를 잘 따르는지 검증
