# CLAUDE.md

## 1. CLAUDE.md 란

- **CLAUDE.md**는 **프로젝트 루트에 두는 설명서**로, 클로드 코드가 새 세션을 열 때 자동으로 읽는 문서
- **프로젝트 컨텍스트/규칙 요약 파일**
- 프로젝트 전반에 걸친 정보를 한 번 입력해 두면 반복 입력이 필요 없다.

그러나 CLAUDE.md는 규칙 파일이 아니라 `컨텍스트 스위처`이다.

- 프로젝트 루트에만 두지 않고, 계층 구조로 활용할 수 있다.

## 2. CLAUDE.md에 포함해야 할 주요 항목

### 1) 기술 스택과 버전

- 프로젝트가 사용하는 프레임워크/라이브러리 + 각각 **정확한 버전**.
- 버전 미기재 시 AI가 알고 있는 안정 버전(오래된 버전)을 사용해 코드 생성할 수 있음.



### 2) 개발 규칙 (Coding Conventions)

- 코드 포맷팅, 변수 네이밍, 테스트 규칙, 스크립트 등
- 스크립트 명시하면 **AI가 자동으로 일관된 스타일을 유지**하게 할 수 있다. 그러므로 배포 관련 스크립트는 제외하는 것이 좋다.


### 3) 개발 워크플로우

- 작성자는 **TDD (Test-Driven Development)** 철학을 반영한 규칙도 추가함.
- 워크플로우를 적어 두면 **AI가 규칙에 맞는 방식으로 코딩**하게 도움.


### 4) 특별 주의사항

- 예: `any` 타입을 금지하거나 특정 코딩 금지 규칙 등
- 이런 세부 지침을 포함하면 **반복적인 실수를 줄일 수 있음**.



## CLAUDE.md를 더 잘 활용하는 법

### 계층적 CLAUDE.md

- 엄청 길게 한 파일에 모두 쓰기보다는 **디렉터리별/하위 모듈별로 나눔**.
- 예: `src/entities/CLAUDE.md` 파일을 두면 그 컨텍스트에서 작업할 때 자동으로 적용됨.
- 이렇게 하면 **컨텍스트가 더 정확하고 혼란이 적음**.


### 컨텍스트 자동 전환

- 예: 인증 + payment + form 관련 작업을 동시에 처리할 때도 각 컨텍스트 규칙을 조합해서 자동으로 적용.
- 결국 Claude Code가 프로젝트 구조를 이해하고 **올바른 규칙 세트를 적용**하는 데 도움이 됨.
